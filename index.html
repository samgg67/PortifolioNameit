1° Entrega - Documento de Visão

O Documento de Visão é um artefato utilizado em projetos de software para definir e comunicar claramente os objetivos, escopo, funcionalidades e requisitos gerais do sistema que será desenvolvido.

2° MVP e Caso de Uso/Diagrama de classe

O MVP (Minimum Viable Product) é a versão mais simples e funcional de um produto, criada com o mínimo de recursos necessários para atender à principal necessidade dos usuários e permitir validação da ideia com o público real.

Os Casos de Uso descrevem, de forma estruturada, como o usuário interage com o sistema para atingir um objetivo específico. São utilizados na análise de requisitos para entender o comportamento funcional do sistema.

3° Diagrama de classes + Workflow

O Diagrama de classes é uma representação visual da estrutura lógica de um banco de dados. Ele mostra como as entidades (como "Cliente", "Pedido", "Produto") se relacionam entre si e quais são seus atributos. É amplamente utilizado na fase de modelagem de dados.

Workflow (fluxo de trabalho) é a definição e automação da sequência de tarefas, atividades ou processos de um sistema ou organização.
